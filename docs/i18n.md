# i18n — Internationalization

## Overview

All user-facing strings live in `src/lib/i18n/`. The active locale is stored in `localStorage` (`podsync-locale`) and can be changed from the **Profile → Language** picker. Switching language updates the entire UI instantly without a page reload.

## File structure

```
src/lib/i18n/
  index.ts              — Locale type, getStrings(), defaultLocale
  LocaleContext.tsx     — LocaleProvider, useStrings(), useLocale() hooks
  locales/
    en.ts               — English strings (default)
    es.ts               — Spanish strings
```

## Adding a new language

1. Create `src/lib/i18n/locales/fr.ts` modelled after `en.ts` — translate every key.
2. In `src/lib/i18n/index.ts`, add `'fr'` to the `Locale` union and import + add the locale to the `locales` map.
3. In `src/lib/i18n/LocaleContext.tsx`, add `fr: 'Français'` to `LOCALE_LABELS`. This is all that's needed — the Profile language picker renders from `LOCALE_LABELS` automatically.

## Using strings in a component

All app pages and components that render user-visible text must use the `useStrings()` hook:

```tsx
import { useStrings } from '@/lib/i18n/LocaleContext'

export default function MyPage() {
  const strings = useStrings()
  return <h1>{strings.queue.heading}</h1>
}
```

Do **not** import the static `strings` export from `index.ts` in client components — it always returns the default locale and won't react to language changes.

## String tone — keep it fun 🎉

User-facing strings (empty states, headings, CTAs) should feel warm and human, not like system messages. Guidelines:

- **Use emojis** in titles and empty-state headings to add personality. Each section has a natural fit:
  - Queue → 🎧
  - History → 📻
  - Search / discover → 🔍 🔎
  - Player / sleep → 💤
  - Subscriptions → 📡
- **Empty states** should tell the user *what to do next*, not just what's missing. "Your queue is empty" is cold; "Your queue is empty 🎧 — add episodes and they'll line up here, ready to play" is helpful and friendly.
- **CTAs** should be action-oriented: "Browse podcasts", "Start listening" — not "Click here" or "Go".
- Keep the same tone in all languages. Don't translate away the emojis.

## String namespaces

| Namespace | Covers |
|-----------|--------|
| `nav` | Sidebar navigation labels, Sign out |
| `queue` | Queue page heading + empty state |
| `history` | History page heading + empty state |
| `sidebar` | "My Podcasts" section + empty hint |
| `profile` | Profile page labels, Account, Language, Subscriptions empty state |
| `discover` | Discover page heading, search UI, Trending, genre tabs, empty states |
| `genres` | Genre tab labels keyed by iTunes genre ID |
| `player` | Playback speed label, sleep timer options, upgrade nudge |
| `guest` | Guest-mode UI: toast, sidebar hint, auth prompt titles, queue sync banner |
| `podcast_page` | Podcast detail page: episode search, guest nudge banner, nav warning modal |
| `welcome_toast` | Returning-user sign-in toast |
| `welcome_toast_new_user` | First-time sign-up toast |

## EmptyState component

`src/components/ui/EmptyState.tsx` is a shared component for empty states across all pages:

```tsx
<EmptyState
  title={strings.queue.empty_title}        // required — shown large, use an emoji
  description={strings.queue.empty_description}  // optional — one sentence of guidance
  cta={{ label: strings.queue.empty_cta, href: '/discover' }}  // optional — link button
/>
```

## Default locale

`defaultLocale` in `src/lib/i18n/index.ts` controls the fallback when no locale is stored in `localStorage` (e.g. first visit). Change it there to shift the app default.
